Тестовый клиент для тестирования запросов RegtimeSRS

Клиент "небезопасен", то есть при отправке через форму запроса с данными (логином/паролем) клиента запрос будет выполнен от его имени. Запросы на всякий случай журналируются. Номер транзакции (в правом поле) является "внутренним" значением этого интерфейса и через API никуда не отправляется.

Пример запроса:
username=test&password=test&utf8=1&interface_revision=1&thisPage=pispRenewDomain&domain_name=berendeevdom.ru&period=1
